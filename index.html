Biblioteca Escolar Digital

Funcional Não Funcional Inverso

Aqui estão os requisitos Funcionais

Aqui vão estar os requisitos

  • O sistema deve ser fácil de usar, com uma interface simples e intuitiva.
  • O sistema deve ser capaz de suportar múltiplos acessos simultâneos sem lentidão.
  • O sistema deve ser acessível em dispositivos móveis, como celulares e tablets.
  • O sistema deve garantir a segurança dos dados dos alunos e professores, utilizando criptografia.
  • Gerenciar todo o acervo de livros, permitindo que os alunos visualizem quais livros estão disponíveis e reservem exemplares com antecedência.
  • Implementar um sistema de lembrete automático para devoluções de livros, evitando que os alunos esqueçam e acumulem atrasos.
  • Permitir que os professores recomendem leituras personalizadas para cada turma.
  • Acompanhar o desempenho dos alunos em relação à leitura.
  • Permitir que os professores registrem sugestões de livros e acessem relatórios sobre os materiais mais utilizados.
  • Fornecer um painel administrativo para monitorar o uso da biblioteca, incluindo estatísticas sobre livros mais lidos, alunos mais ativos e frequência de utilização da plataforma.
  • Garantir que apenas alunos e professores da escola tenham acesso ao sistema, evitando o uso indevido por pessoas externas.
  • O sistema deve permitir que os alunos verifiquem quais livros estão disponíveis antes de irem à biblioteca.
  • O sistema deve permitir que os alunos reservem livros antecipadamente.
  • O sistema deve enviar notificações para os alunos quando a data de devolução estiver próxima.
  • O sistema deve disponibilizar livros no formato digital para que os alunos possam acessá-los de qualquer lugar (celular, tablet, etc.).
  • O sistema deve sugerir livros com base nas leituras anteriores dos alunos.
  • O sistema deve ter um ranking dos livros mais lidos, incentivando os alunos a lerem mais.
  • O sistema deve incluir um espaço para discussões sobre os livros, permitindo que os alunos compartilhem suas opiniões.


Aqui estão os requisitos Não Funcionais

Aqui vão estar os requisitos

  • O sistema não deve permitir que muitos alunos acessem os livros ao mesmo tempo, causando lentidão ou falhas.
  • O sistema não deve permitir que os livros fiquem disponíveis por tempo excessivo, impedindo a rotatividade entre os alunos.
  • O sistema não deve permitir a inclusão de atividades complementares irrelevantes ou desconexas dos livros recomendados.
  • O sistema não deve armazenar dados dos alunos sem as devidas medidas de segurança, como criptografia.
  • O sistema deve ser acessível e eficiente, melhorando a experiência de alunos e professores no processo de ensino e aprendizagem.
  • O sistema deve ser acessível em dispositivos móveis, como celulares e tablets.
  • O sistema deve ser fácil de usar e intuitivo, permitindo uma experiência agradável para os alunos.
  • O sistema deve ter um design que incentive o uso contínuo e a interação dos alunos.


Aqui estão os requisitos Inversos

Aqui vão estar os requisitos

  • Os alunos não devem poder acessar os livros recomendados diretamente sem passar pelo processo de autenticação.
  • O sistema não deve permitir a recomendação de livros irrelevantes para os alunos, com base em seu desempenho ou preferências.
  • O sistema não deve permitir que os alunos acessem múltiplos livros ao mesmo tempo sem restrição.
  • Evitar que livros sejam extraviados ou devolvidos com atraso, prejudicando outros alunos que precisam daquele material.
  • Evitar que pessoas externas tenham acesso ao sistema, garantindo que apenas alunos e professores da escola possam utilizar os recursos.
  • O sistema não deve dificultar o processo de reserva de livros, ou seja, não pode ser complicado ou demorado.
  • O sistema não deve permitir o acesso aos livros de forma desorganizada, causando confusão entre os alunos.
  • O sistema não deve deixar de enviar as notificações de devolução, o que poderia causar atrasos e problemas para os alunos.
